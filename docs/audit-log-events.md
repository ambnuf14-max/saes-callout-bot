# Audit Log — события

Все события логируются в Discord-канал, заданный через `/settings → Audit Log`.
Каждый embed содержит **footer** с именем и ID пользователя, **timestamp** и опциональный **thumbnail** (иконка подразделения/фракции).

---

## Текущие события (39 типов)

### 🚨 Каллауты

| Событие | Enum | Поля |
|---|---|---|
| **Каллаут создан** | `callout_created` | ID каллаута, подразделение, фракция, канал, создатель, кратко об инциденте, статус VK/TG уведомлений |
| **Каллаут закрыт** | `callout_closed` | ID каллаута, подразделение, кто закрыл (mention или «Система»), канал, длительность, причина |
| **Каллаут закрыт по таймауту** | `callout_auto_closed` | ID каллаута, подразделение, «Система (автотаймаут)», канал, длительность |

> `callout_auto_closed` срабатывает вместо `callout_closed` когда каллаут закрывается автоматически по `CALLOUT_AUTO_CLOSE_DELAY`. Позволяет видеть инциденты, на которые никто не отреагировал.

### ✅ Система одобрения изменений (pending)

| Событие | Enum | Поля |
|---|---|---|
| **Запрос на создание подразделения** | `subdivision_create_requested` | ID запроса, тип, фракция, кто запросил, детали diff |
| **Запрос на обновление подразделения** | `subdivision_update_requested` | ID запроса, тип, фракция, кто запросил, детали diff |
| **Запрос на удаление подразделения** | `subdivision_delete_requested` | ID запроса, тип, фракция, кто запросил, детали diff |
| **Запрос на обновление embed** | `embed_update_requested` | ID запроса, тип, фракция, кто запросил, детали diff |
| **Изменение одобрено** | `change_approved` | Тип, фракция, кто одобрил, кто запросил, детали |
| **Изменение отклонено** | `change_rejected` | Тип, фракция, кто отклонил, причина, детали |
| **Изменение отменено** | `change_cancelled` | Тип, фракция, детали |

> Запросы отправляются с кнопками **Одобрить / Отклонить** прямо в audit log канал (`logPendingChangeWithButtons`). После принятия решения сообщение редактируется — кнопки убираются, добавляется информация о решении.
