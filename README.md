# SAES Callout Bot

Discord & VK интеграция для системы каллаутов (оповещений о вызовах) для ролевого сервера Los Santos.

## Описание

Система автоматизирует процесс создания и обработки каллаутов экстренных служб:

- **Discord**: Пользователи создают каллауты через интерактивную панель, система создает выделенный канал с деталями инцидента
- **VK**: Автоматическая отправка уведомлений в беседы департаментов с кнопкой реагирования
- **Синхронизация**: При реагировании из VK отправляется подтверждение в Discord канал

## Возможности

- ✅ Создание каллаутов через Discord с модальными окнами
- ✅ Автоматическое создание каналов с правами доступа
- ✅ Уведомления в VK беседы с кнопками реагирования
- ✅ Синхронизация ответов VK → Discord
- ✅ Закрытие инцидентов автором или департаментом
- ✅ Гибкая система департаментов
- ✅ Панель управления для лидеров (Discord команды)
- ✅ История каллаутов в SQLite БД

## Установка

1. **Клонировать репозиторий**
   ```bash
   cd path/to/saes-callout-bot
   ```

2. **Установить зависимости**
   ```bash
   npm install
   ```

3. **Настроить переменные окружения**
   ```bash
   cp .env.example .env
   ```

   Отредактировать `.env` и заполнить:
   - `DISCORD_TOKEN` - токен Discord бота
   - `DISCORD_CLIENT_ID` - ID Discord приложения
   - `VK_TOKEN` - токен VK группы
   - `VK_GROUP_ID` - ID VK группы

4. **Запустить миграции базы данных**
   ```bash
   npm run migrate
   ```

5. **Запустить в режиме разработки**
   ```bash
   npm run dev
   ```

   Или собрать и запустить в production:
   ```bash
   npm run build
   npm start
   ```

## Настройка Discord бота

1. Перейти на [Discord Developer Portal](https://discord.com/developers/applications)
2. Создать новое приложение
3. Во вкладке "Bot":
   - Создать бота
   - Скопировать токен → `DISCORD_TOKEN`
   - Включить Intents:
     - ✅ Presence Intent
     - ✅ Server Members Intent
     - ✅ Message Content Intent
4. Во вкладке "OAuth2":
   - Скопировать Client ID → `DISCORD_CLIENT_ID`
   - В "OAuth2 URL Generator" выбрать:
     - Scopes: `bot`, `applications.commands`
     - Bot Permissions: `Administrator` (или настроить детально)
   - Использовать сгенерированную ссылку для добавления бота на сервер

## Настройка VK бота

1. Создать VK группу
2. Перейти в "Управление" → "Работа с API"
3. Создать токен с правами:
   - ✅ Сообщения сообщества
   - ✅ Управление сообществом
4. Скопировать токен → `VK_TOKEN`
5. ID группы можно найти в настройках → `VK_GROUP_ID`
6. Включить Long Poll в настройках "Сообщения"
7. Добавить бота в беседы департаментов

## Первый запуск

После запуска бота:

1. **Начальная настройка** (только администраторы):
   ```
   /setup
   ```
   Создаст категорию "🚨 INCIDENTS" и канал "callouts" с кнопкой

2. **Добавить департаменты**:
   ```
   /department add LSFD @LSFD 2000000001
   ```
   - `LSFD` - название департамента (2-10 символов, заглавные)
   - `@LSFD` - роль Discord для упоминания
   - `2000000001` - ID VK беседы (peer_id)

3. **Список департаментов**:
   ```
   /department list
   ```

4. **Удалить департамент**:
   ```
   /department remove LSFD
   ```

## Использование

### Создание каллаута

1. Пользователь заходит в канал #callouts
2. Нажимает кнопку "📞 Создать каллаут"
3. В модальном окне выбирает департамент и описывает инцидент
4. Система:
   - Создает канал `incident-{id}-{department}`
   - Отправляет embed с информацией
   - Упоминает роль департамента
   - Отправляет уведомление в VK беседу

### Реагирование из VK

1. Участник департамента в VK беседе видит уведомление
2. Нажимает кнопку "📞 Отреагировать на инцидент"
3. Система отправляет подтверждение в Discord канал инцидента

### Закрытие каллаута

В канале инцидента:
```
/close [причина]
```

Могут закрывать:
- Автор каллаута
- Члены департамента
- Лидеры департаментов

## Команды Discord

- `/setup` - Начальная настройка системы (только администраторы)
- `/department add <name> <role> <vk_chat>` - Добавить департамент
- `/department list` - Список департаментов
- `/department remove <name>` - Удалить департамент
- `/close [reason]` - Закрыть текущий инцидент

## База данных

SQLite с 4 таблицами:
- `servers` - настройки Discord серверов
- `departments` - департаменты (LSFD, LSPD, EMS, и т.д.)
- `callouts` - инциденты/каллауты
- `callout_responses` - ответы департаментов

## Логирование

Логи сохраняются в папке `logs/`:
- `bot.log` - все логи
- `error.log` - только ошибки

В режиме разработки логи также выводятся в консоль с цветами.

## Docker

### Запуск бота в Docker

```bash
# Запустить контейнер (в фоне)
npm run docker:up
# или
docker-compose up -d

# Остановить контейнер
npm run docker:down
# или
docker-compose down

# Перезапустить контейнер
npm run docker:restart
# или
docker-compose restart
```

### Просмотр логов Docker

```bash
# Показать последние 1000 строк логов + следить в реальном времени
npm run docker:logs

# Показать ВСЕ логи (с самого начала) + следить в реальном времени
npm run docker:logs:all

# Или вручную:
docker-compose logs -f --tail=1000    # Последние 1000 строк
docker-compose logs -f --tail=all     # ВСЕ логи
docker-compose logs -f                # Только новые логи
```

**Важно**: Docker хранит до 500MB логов (10 файлов по 50MB). Старые логи автоматически ротируются.
